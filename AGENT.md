# AGENT.md

> 本文件为 AI 编码助手（Agent）提供在本项目中工作的指引。
> 项目目标：构建一个受 [ozgur.design](https://ozgur.design/) 启发的、极简编辑式的个人作品集网站。
> 配套文档：设计语言见 `design.md`，可复用的实现技能见 `SKILL.md`。

---

## 1. 项目概览

- **类型**：个人作品集 / 个人介绍网站。
- **风格**：Bauhaus 式极简、编辑排版（editorial）、近黑白配色、形式追随功能。
- **技术栈**：Astro + Tailwind CSS，内容用 Markdown 编写，部署到 Vercel。
- **首要原则**：**意图与产物零偏差** —— 所写即所得，不引入隐藏复杂度。

---

## 2. 黄金法则（Agent 必须遵守）

1. **克制优先**：能不加的依赖就不加，能不写的 JS 就不写。每个新增物都要「值其重量」。
2. **静态优先**：默认产出静态 HTML/CSS；JS 仅用于渐进增强。
3. **可访问性不可妥协**：语义化标签、键盘可达、遵守 `prefers-reduced-motion` 与 `prefers-color-scheme`。
4. **遵循设计规范**：所有视觉决策须对照 `design.md`，不要自创风格。
5. **小步提交**：每次改动聚焦单一目标，保持 diff 可读。
6. **不破坏构建**：改动后确保 `build` 通过、无 lint 错误。

---

## 3. 代码风格约定

- **语言**：TypeScript 优先；样式使用 Tailwind 原子类。
- **组件**：保持单一职责、可组合；避免过度抽象。
- **命名**：语义化、英文小驼峰 / 短横线（文件名）；编号章节用 `00`–`10` 命名约定。
- **注释**：仅解释「为什么」，不复述「做了什么」。
- **格式**：交由格式化工具（Prettier / Tailwind 排序插件）统一，不手工争论空格。

---

## 4. 排版与色彩（速查，详见 design.md）

- 等宽字体用于编号 / 标签 / 元信息；阅读体字族用于标题 / 正文。
- 章节标题：单词 + 句号（如 `Approach.`）。年份用罗马数字。
- 配色限定在「近白背景 / 近黑文字 / 中灰次级」，强调色仅用于交互态。
- 用留白分隔内容，不用分割线和卡片堆叠。

---

## 5. 常用命令

> 实际命令以 `package.json` 为准，以下为约定占位。

```bash
# 安装依赖
pnpm install

# 本地开发
pnpm dev

# 生产构建
pnpm build

# 预览构建产物
pnpm preview

# 代码检查
pnpm lint

# Prettier 格式化
pnpm format
```

---

## 6. 工作流程建议

1. **先读后写**：动手前先读 `design.md` 与相关现有文件。
2. **对照规范**：实现时逐条核对 `design.md` 的「设计检查清单」。
3. **渐进增强**：先保证无 JS 可用，再叠加动效。
4. **自检**：构建通过、无 lint 报错、`prefers-reduced-motion` 下动画暂停。
5. **复用技能**：重复性实现模式记录并复用 `SKILL.md`。

---

## 7. 禁止事项（Don'ts）

- ❌ 不引入重型 UI 库或与极简风格冲突的组件库。
- ❌ 不堆叠装饰性动画、视差、自动播放媒体。
- ❌ 不使用低对比配色或纯靠颜色传达信息。
- ❌ 不在没有降级方案的情况下强依赖客户端 JS。
- ❌ 不偏离 `design.md` 自行发挥风格。

---

## 8. 完成定义（Definition of Done）

- [ ] 视觉与 `design.md` 一致。
- [ ] 构建成功、无 lint / 类型错误。
- [ ] 禁用 JS 时页面完整可读。
- [ ] 通过基本可访问性自查（对比度、键盘、reduced-motion）。
- [ ] 改动聚焦、提交信息清晰。
