# 技能：编辑式极简作品集（Editorial Minimal Portfolio）

本技能指导如何实现一个像 [ozgur.design](https://ozgur.design/) 那样的、以**排版与留白为主角**、形式追随功能的个人作品集网站。配合 `design.md`（设计语言）与 `AGENT.md`（工程约定）一起使用。

---

## 何时使用

- 用户要做**个人介绍 / 作品集 / portfolio**站点。
- 需要**编号章节（00–10）的长页叙事**布局。
- 想要**极简、编辑式、近黑白、Bauhaus** 的视觉风格。
- 在已有站点中**新增一个章节**或一篇文章。

---

## 核心原则（务必内化）

1. **形式追随功能**：不服务读者的功能不做，不值其重量的依赖不加。
2. **静态优先 + 渐进增强**：默认静态 HTML/CSS，JS 仅为锦上添花。
3. **排版即设计**：用字号、字族、留白、对齐建立层级，而非颜色与线框。
4. **意图与产物零偏差**：所写即所得。

---

## 实现步骤

### 第 1 步：建立排版与色彩基底

- 引入**双字族**：等宽字体（标签 / 编号 / 元信息）+ 阅读体（标题 / 正文）。
- 定义近黑白配色变量：`背景(近白)`、`前景(近黑)`、`次级(中灰)`、`强调(仅交互态)`。
- 设定正文 measure（约 60–75 字符）与垂直节奏（行距、段距）。

### 第 2 步：搭建编号章节骨架

每个章节遵循统一模式：

```html
<section id="03-background" aria-labelledby="03-title">
  <p class="meta">03 / Background</p>
  <!-- 等宽标签 -->
  <h2 id="03-title">Beginnings.</h2>
  <!-- 单词标题 + 句号 -->
  <div class="prose">…正文…</div>
</section>
```

- 编号格式：`两位数字 / 关键词`。
- 标题：单个有力的词 + 句号。
- 年份用罗马数字（如 `MMXXVI`）。

### 第 3 步：内容用 Markdown 驱动

- 章节正文、文章、项目列表均以 Markdown 编写，由 Astro 渲染为扁平静态页。
- 列表项采用「名称 + 一句话描述」的双栏 / 紧凑排布。

### 第 4 步：克制地添加动效

- 仅做轻量入场（淡入 / 微位移），使用 GSAP 或 Motion。
- 必须包裹 `prefers-reduced-motion` 判断：开启减少动态时**暂停全部动画**并给出提示。

### 第 5 步：可访问性与降级

- 语义化标签、正确标题层级、清晰焦点态。
- 禁用 JS 时页面完整可读。
- 遵守 `prefers-color-scheme` 与 WCAG AA 对比度。

---

## 推荐技术栈

- 框架：**Astro**　样式：**Tailwind CSS**　内容：**Markdown**
- 动效：GSAP / Motion（可选）　组件：必要处 React / shadcn
- 部署：Vercel / Cloudflare

---

## 验收清单（完成前逐条核对）

- [ ] 每个章节都有「编号 + 单词标题（句号）」。
- [ ] 使用等宽标签 + 阅读体正文的双字族体系。
- [ ] 配色克制在近黑白 + 中灰，强调色仅用于交互。
- [ ] 用留白而非线框 / 卡片分隔内容。
- [ ] 动效遵守 `prefers-reduced-motion`。
- [ ] 禁用 JS 时站点完全可读可用。
- [ ] 无多余依赖、无装饰性元素（form follows function）。

---

## 反模式（避免）

- ❌ 堆叠卡片、阴影、渐变等装饰。
- ❌ 重型 UI 库、抢注意力的动画、自动播放媒体。
- ❌ 低对比配色、纯靠颜色传达信息。
- ❌ 强依赖客户端 JS 而无降级方案。
- ❌ 偏离 `design.md` 自创风格。
